Approach C — Thread Colour Detail Popover

Shown for DMC 321 (Christmas Red) and blanc (White). The popover opens when the user clicks a colour swatch anywhere in the app.

In production this is rendered via ReactDOM.createPortal() to document.body, positioned with position: fixed. The arrow at the top points to the swatch that was clicked. Closes on Escape, click outside, or the X button.

Positioning logic (production): The popover is positioned with position: fixed, anchored below the swatch's getBoundingClientRect().bottom and aligned to its left edge. If it would overflow the bottom of the viewport, it flips above. If it would overflow the right edge, it clamps to viewport.width - popoverWidth - 8px. On viewports narrower than 480px it renders as a bottom sheet instead (see mobile section below).

Design note: The large 48×48px swatch in the popover is deliberately flat (no texture gradient), to serve as a colour reference. It is larger than the palette list swatch to make the colour itself the focus. The three fabric preview chips are pure CSS — no canvas rendering required. The fabric background is the outer square; the thread colour is the centred inner chip.

Mobile — bottom sheet layout (viewport < 480px)

On mobile, the popover renders as a bottom sheet anchored to the viewport bottom. The same content is shown; the layout adapts to a wider single-column format.
