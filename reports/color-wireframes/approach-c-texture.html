Approach C — Texture Simulation: Swatch Comparison

Three rendering modes across 20 DMC threads at two sizes. Toggle between light and dark backgrounds to verify the overlay is neutral.

Background simulates the app's Workshop theme
CSS approach. Because swatch background colour is set via an inline background shorthand (which overrides background-image), the texture must be applied via a ::after pseudo-element with position: absolute; inset: 0. This wireframe uses a wrapper div to simulate the same visual output. In production, the rule lives in styles.css on .colour-swatch::after and requires position: relative; overflow: hidden on .colour-swatch.
The "Over-strong" column is a warning example only — do not ship values above 20% white opacity. At 55% the highlight becomes a visual artefact that competes with the hue.

Production CSS (styles.css additions)

/* Add to existing .colour-swatch block */ .colour-swatch { /* ... existing rules ... */ position: relative; overflow: hidden; /* clips ::after to rounded corners */ } .colour-swatch::after { content: ''; position: absolute; inset: 0; border-radius: inherit; background: linear-gradient( 135deg, rgba(255, 255, 255, 0.18) 0%, transparent 50%, rgba(0, 0, 0, 0.06) 100% ); pointer-events: none; } /* Reduced opacity for the 40×60px tall swatches in the comparator */ .colour-swatch--large::after { background: linear-gradient( 135deg, rgba(255, 255, 255, 0.14) 0%, transparent 50%, rgba(0, 0, 0, 0.05) 100% ); }

Summary: At 18% white / 6% black, the gradient overlay is within the perceptibility threshold for all tested colours. No swatches show a hue shift that would cause a colour identification error. The most visible case (black, DMC 310) reads correctly as thread specularity. Light pastel blues (827) should be monitored at the 40×60px comparator swatch size — the .colour-swatch--large modifier at 14% / 5% is recommended for that use.
