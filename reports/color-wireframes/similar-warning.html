Similar Colour Warning — UI Wireframe

How the similar-colour warning would appear in the palette list. Triggered when two palette colours have ΔE₀₀ < 3.0.

This warning addresses a real user problem: selecting two threads that look almost identical on screen, purchasing both, and discovering they are nearly indistinguishable in the finished work. The warning does not prevent the user from keeping both — it flags the issue for their attention.

Scenario: User has blanc and B5200 in their palette

With the current data, blanc and B5200 are identical (ΔE₀₀ = 0). After the data fix, they will be ΔE₀₀ ≈ 1.8 — still triggering the warning (threshold: 3.0). This is correct: these two threads are genuinely very similar and users should verify.

Design notes

  • Warning dot: small (8px), amber/orange. Visible but not alarming.
  • Tooltip: shown on hover. Not inline — too verbose if shown for all warning rows simultaneously.
  • Threshold: ΔE₀₀ < 3.0 triggers warning. This is "just perceptible to clearly perceptible" range.
  • Does not block the user — they can keep both colours. It is advisory only.
  • Does not appear for intentionally similar shading sequences (user can dismiss per-pair).
  • Warning icon/dot must use window.Icons SVGs, not emoji or Unicode symbols.
