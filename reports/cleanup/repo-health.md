## Findings

### I0-1 · `.git` is 99 MB of loose objects — `git gc` recovers ~90 MB

**Evidence:**
```
git count-objects -vH  →  size: 98.99 MiB, in-pack: 1136, packs: 2,
                          size-pack: 7.73 MiB, prune-packable: 402
.git/objects loose:  99.0 MB
.git/objects/pack:    7.7 MB
```

**Diagnosis:** Recent commits (the audit / cleanup churn visible in shell history) created thousands of loose objects that have not been repacked. `prune-packable: 402` means hundreds of objects exist *both* loose and in a pack — pure waste. There is no `git auto-gc` budget breach yet because the loose-object count threshold is high, but the byte cost is already large.

**Recommendation (I0):**
```
git gc --prune=now
# or, more aggressive (one-time):
git gc --aggressive --prune=now
```
Expected result: `.git` drops from 107 MB to ~10–15 MB. Zero risk: no history is rewritten, no collaborator action is required, no branches change SHA. Repeat after every large cleanup pass.

**Measurement:** `Get-ChildItem .git -Recurse -File | Measure-Object -Sum Length` before/after.

### I1-1 · 249 historical `creator/bundle.js` blobs — consider history rewrite carefully

**Evidence:** History contains **249 distinct `creator/bundle.js` blobs** averaging 590 KB, totalling **~143 MB uncompressed** in object DB. They compress well (which is why pack is only 7.7 MB) but each new bundle commit ratchets the loose-object count.

| Top history blobs (KB) | Path |
|---:|---|
| 20,457 | `Books and Blossoms - 5mm color.pdf` (no longer tracked, only in history) |
| 8,077 | `TestUploads/Books and Blossoms.pdf` |
| 2,624 | `assets/Books and Blossoms - BW.svg` |
| 1,784 | `TestUploads/exported_pattern.pdf` |
| 1,385 | `TestUploads/PAT2171_2.pdf` |
| 1,062 | `pdf.worker.min.js` |
| 854 | `creator/bundle.js` (current; 249 historical revisions) |

**Recommendation:** Do **not** run `git filter-repo` / BFG yet. The pack is already compact (7.7 MB), so a history rewrite recovers little disk and forces every collaborator to re-clone. Defer until either:
- The repo is being archived/forked and clone size matters acutely, **or**
- Multiple `>10 MB` blobs accumulate in the future.

If/when rewrite happens, the high-value targets are: `Books and Blossoms - 5mm color.pdf` (20 MB single blob), all historical `creator/bundle.js` revisions, and any `TestUploads/*.pdf` that get deleted by I1-3.

**Cost of rewrite:** Every clone, fork, and CI cache invalidates. Open PRs from origin's 61 remote-tracking branches would need rebasing. **Worth it only if `.git` exceeds ~50 MB packed.** Today: not worth it.

### I1-2 · Stop committing generated artifacts

**Tracked generated files:**
- [creator/bundle.js](../../creator/bundle.js) — 854 KB; output of `node build-creator-bundle.js`
- [creator/import-wizard-bundle.js](../../creator/import-wizard-bundle.js) — 30 KB; output of `node build-import-bundle.js`
- [import-engine/bundle.js](../../import-engine/bundle.js) — also generated
- [assets/fonts/CrossStitchSymbols.base64.js](../../assets/fonts/CrossStitchSymbols.base64.js) — output of `node build-symbol-font.js`

**Tradeoff:** the project is intentionally build-step-free at runtime (browsers load bundles via `<script>`). **If `creator/bundle.js` is required for the production deployment to work without a build step on the host (Vercel)**, then either:
- (a) keep tracking but treat as a generated lockfile-equivalent and gate on a `build:check` script in CI, **or**
- (b) add a `vercel-build` step (`node build-creator-bundle.js && node build-import-bundle.js`) and remove from tracking.

**Verify before changing:** `cat vercel.json` to see if there's already a build hook. If Vercel just serves the repo verbatim with no build step, option (a) is the only safe choice; do **not** untrack `creator/bundle.js` until that's confirmed.

**Action regardless of (a)/(b):** add `.gitattributes`:
```
creator/bundle.js                    linguist-generated=true -diff
creator/import-wizard-bundle.js      linguist-generated=true -diff
import-engine/bundle.js              linguist-generated=true -diff
assets/fonts/CrossStitchSymbols.base64.js linguist-generated=true -diff
pdf.worker.min.js                    linguist-generated=true -diff
assets/fontkit.umd.min.js            linguist-generated=true -diff
* text=auto eol=lf
*.png binary
*.pdf binary
*.ttf binary
*.svg text
```
This stops GitHub from showing diff churn / language-stats noise for these files. No history change, no risk.

### I2-2 · Add `.editorconfig`

No `.editorconfig` exists. With Windows + Unix collaborators and a mix of minified-style (`constants.js`) and modern (`creator/*.js`) files, line-ending drift is a real risk. Recommended:

```ini
# .editorconfig
root = true

[*]
charset = utf-8
end_of_line = lf
insert_final_newline = true
trim_trailing_whitespace = true
indent_style = space
indent_size = 2

[*.md]
trim_trailing_whitespace = false

[Makefile]
indent_style = tab
```

Pair with the `* text=auto eol=lf` line in `.gitattributes` from I1-2.
