Stash-Adapt — Wireframes

Three approaches, each as a single self-contained HTML page. Every page renders the entry point, the main review screen (desktop + mobile), the per-row substitution picker, the original/adapted preview state, and the confirmation/done state.

  • Approach A — Review Table (recommended)
  • Approach B — Pattern-First Preview
  • Approach C — Guided Wizard
Wireframes use realistic DMC names/numbers, a mix of good/fair/poor/no-match outcomes, and the recommended match-quality vocabulary from reports/stash-adapt-8-match-quality-ui.md. Visual style is deliberately neutral — final visuals will use the Workshop tokens from styles.css.
