What this option eliminates
✕
manager.html and stitch.html as "landing pages" — clicking Stash or Track from the home page used to open those full pages even with no project context, creating a secondary landing experience. Now Stash is a tab on home; Tracker is only opened once you've chosen a project.
✕
index.html?action=new-from-image as an entry point — the creator page is no longer a landing page. "Create new" is a tab on home; the creator only opens once a project is being actively worked on.
✕
QuickTiles "Open Tracker" + "Open Stash Manager" — those are now tabs, not tiles. No duplicate destinations.
✕
GreetingHero + ResumeCard split — one resume card at the top of the Projects tab covers both purposes.
✕
home-screen.js legacy component — the legacy home screen inside the Creator is redundant. This approach makes home.html the definitive landing and the Creator only opens for editing/importing.
